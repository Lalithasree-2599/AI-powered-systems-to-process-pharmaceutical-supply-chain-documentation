# Advanced AI-Powered Pharmaceutical Document Processing

This repository contains all project files of the **Pfizer Supply Chain Document Processing Externship**.  
The program focuses on building an **end-to-end AI pipeline for pharmaceutical document automation** using **OCR, NLP, LLMs, Python, and Retrieval-Augmented Generation (RAG)**.

The goal is to automate extraction and analysis of complex pharmaceutical documents such as:

- Certificates of Analysis (CoA)
- Safety Data Sheets (SDS)
- Product Specifications
- Compliance Statements
- Supply Chain Traceability Documents

---

## 📚 Project Overview

### ✅ Project 1 — Working With Python for AI Document Processing
Introduction to Python tools used in document automation.

Topics include:
- Python fundamentals for data processing
- Working with PDFs and document files
- Data cleaning and preprocessing
- Preparing document data for AI pipelines

➡️ **Outcome:** Ability to process and prepare document datasets for AI workflows.

---

### ✅ Project 2 — Data Extraction from Documents Using Python
Build rule-based and programmatic extraction pipelines.

Tasks include:
- Extracting product names, batch numbers, and manufacturing data
- Parsing tables from pharmaceutical documents
- Using regex and structured extraction logic
- Converting document content into structured datasets

➡️ **Outcome:** Automated extraction of structured data from pharmaceutical documents.

---

### ✅ Project 3 — Advanced OCR Comparison & Layout-Aware Extraction
Evaluate OCR engines for pharmaceutical documentation.

OCR engines compared:
- **Tesseract**
- **PaddleOCR**
- **EasyOCR**

Tasks include:
- Text extraction from scanned regulatory documents
- Handling tables and complex layouts
- Benchmarking OCR accuracy and performance

➡️ **Outcome:** Ability to select and optimize OCR solutions for document processing pipelines.

---

### ✅ Project 4 — Introduction to Retrieval-Augmented Generation (RAG)
Build a RAG pipeline to enable intelligent document search.

Topics include:
- Embedding generation
- Document chunking strategies
- Vector search and retrieval
- Integrating LLMs with document databases

➡️ **Outcome:** AI system capable of retrieving relevant information from large document sets.

---

### ✅ Project 5 — Advanced RAG with Open-Source Models
Improve the RAG pipeline with advanced techniques.

Enhancements include:
- Metadata filtering
- Improved chunking strategies
- Experimenting with open-source LLMs such as:
  - **Mistral**
  - **Phi**
  - **Llama models**

➡️ **Outcome:** Efficient and scalable document retrieval and question answering.

---

### ✅ Project 6 — Blob Processing, Classification, and Routing
Process multi-document PDF bundles.

Tasks include:
- Splitting large "blob" PDFs into individual documents
- Classifying pharmaceutical document types
- Routing documents to the correct extraction pipeline

Example document types:
- Certificate of Analysis
- Safety Data Sheet
- Product Specification

➡️ **Outcome:** Automated document classification and workflow routing.

---

### ✅ Project 7 — RAG Chatbot with Gradio Interface
Build a chatbot that interacts with the document pipeline.

Features include:
- Question answering over pharmaceutical documents
- Interactive chatbot UI using **Gradio**
- Support for multiple LLMs

Example queries:
- "What is the batch number in this certificate of analysis?"
- "What hazards are listed in the safety data sheet?"
- "What is the retest date for this product?"

➡️ **Outcome:** User-friendly interface for querying regulatory documents.

---

### ✅ Project 8 — Final End-to-End Integration
Combine all components into a complete AI-powered document automation system.

Pipeline includes:
- OCR processing
- Data extraction
- Document classification
- Vector search
- Retrieval-Augmented Generation
- Chatbot interface

Tasks include:
- Testing pipeline accuracy and performance
- Evaluating retrieval quality
- Final system demonstration

➡️ **Outcome:** A complete **AI-powered Pharmaceutical Document Automation Platform**.
